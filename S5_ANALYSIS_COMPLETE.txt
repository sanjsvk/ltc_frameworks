KEY INSIGHTS

1. MCMC FLEXIBILITY ADVANTAGE (Confirmed)
   - Only MCMC (among F3 models) can recover weak-signal scenario
   - Joint optimization of decay, ltc_coef, stock_init enables identification
   - Bayesian posterior adapts all parameters to data likelihood simultaneously

2. FIXED-DECAY MODELS ARE BRITTLE (Confirmed)
   - kalman_dlm & bsts fail even with:
     * Weakened decay priors (0.72 vs 0.90 for TV)
     * Reduced stock initialization (60-75% lower)
     * True observation noise (obs_var = 0.09)
   - Problem is NOT noise variance, but inability to adapt DECAY structure
   - These models require decay to be specified upfront; cannot optimize it from data

3. SIGNAL STRENGTH IS A CALIBRATION BOUNDARY (NOT STRUCTURAL)
   - S5 frozen params (0%) → supplementary params (88.5%) shows same framework, 88.5pp difference
   - Practitioners cannot use frozen parameters below signal threshold
   - Must either:
     a) Assess signal strength upfront & select priors accordingly, OR
     b) Use adaptive methods (Bayesian MCMC) that optimize all parameters jointly

4. OBS_VAR SPECIFICATION IS RED HERRING
   - Adding true noise variance (0.09) to kalman_dlm & bsts: NO IMPROVEMENT
   - Bottleneck is NOT noise estimation, but structural inflexibility
   - For future work: kalman_dlm/bsts would need adaptive decay mechanism to handle weak signal

ANSWER TO ORIGINAL RESEARCH QUESTION

Q: "Why do all frameworks fail on S5?"

A: NOT because signal is structurally unrecoverable, but because frozen parameters misalign
   with weak-signal regime. MCMC can recover 88.5% with proper calibration.

Q: "Is this a framework limitation?"

A: NO. It's a calibration boundary. MCMC's Bayesian flexibility enables joint parameter
   optimization; fixed-decay models lack this mechanism and cannot adapt below signal threshold.

Q: "What should practitioners do?"

A: Assess signal strength upfront. If weak, use MCMC with informative priors rather than
   frozen-parameter methods. This finding argues for MCMC as default in production systems.
